# 6. Como Contribuir

Ficamos felizes com o seu interesse em contribuir para o **Assistente de Conteúdo com IA**! Toda ajuda é bem-vinda, seja reportando bugs, sugerindo funcionalidades ou enviando código.

### Reportando Bugs (Issues)

Se você encontrar um bug, por favor, abra uma [**nova Issue**](https://github.com/Yelllowww/Projeto_Integrador/issues) e forneça o máximo de detalhes possível:
-   **Passos para reproduzir**: Descreva exatamente como o erro aconteceu.
-   **Comportamento esperado**: O que você esperava que acontecesse.
-   **Comportamento atual**: O que de fato aconteceu.
-   **Capturas de tela**: Se possível, inclua imagens do erro.

### Sugerindo Melhorias

Se você tem uma ideia para uma nova funcionalidade ou uma melhoria, sinta-se à vontade para abrir uma **Issue** com a sua sugestão.

### Contribuindo com Código (Pull Requests)

1.  **Faça um Fork** do repositório para a sua conta do GitHub.
2.  **Crie uma Branch** para a sua modificação:
    ```bash
    git checkout -b feature/nome-da-sua-feature
    ```
3.  **Faça as alterações** e realize os commits. Escreva mensagens de commit claras e descritivas.
    ```bash
    git commit -m "feat: Adiciona nova funcionalidade X"
    ```
4.  **Envie as alterações** para o seu fork:
    ```bash
    git push origin feature/nome-da-sua-feature
    ```
5.  **Abra um Pull Request (PR)** a partir da sua branch para a branch `main` do repositório original. No PR, descreva detalhadamente as alterações que você fez.

Aguarde a revisão do seu PR. Podemos solicitar algumas alterações antes de fazer o merge. Obrigado por sua contribuição!
